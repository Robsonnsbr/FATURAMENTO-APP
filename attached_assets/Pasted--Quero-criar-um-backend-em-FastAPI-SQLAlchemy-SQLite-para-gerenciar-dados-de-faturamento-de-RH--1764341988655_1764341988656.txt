“Quero criar um backend em FastAPI + SQLAlchemy + SQLite para gerenciar dados de faturamento de RH.
Crie os modelos com base na seguinte descrição:

company(id, name, cnpj_femsa, created_at)

unit(id, company_id, cnpj_unidade, nome_unidade, centro_custo_femsa, created_at)

employee(id, cpf, nome, created_at)

employment_contract(id, employee_id, company_id, unit_id, cargo, funcao, salario_base, data_admissao, data_demissao, data_termino_contrato)

billing_period(id, company_id, mes_referencia, status, created_at)

payroll_item_type(id, code, description, "group", direction)

payroll_item(id, billing_period_id, employee_id, contract_id, unit_id, payroll_item_type_id, quantity, amount, source_column, notes)

exam_record(id, billing_period_id, unit_id, employee_id, tipo, exame, data_pedido, data_exame, data_inativacao, valor_cobrar)

Em seguida:

Crie endpoints para:

upload de um arquivo CSV/Excel de folha (primeira planilha) para um billing_period.

upload de um arquivo CSV/Excel de exames (segunda planilha).

No upload de folha:

Para cada linha, crie/associe company, unit, employee, employment_contract e billing_period.

Crie vários registros em payroll_item mapeando as colunas:

'SALARIO DIA (Qtde)' + 'SALARIO DIA (Valor)' -> payroll_item_type code='SALARIO_DIA'

'HORAS EXTRAS (Qtde)' + 'HORAS EXTRAS (Valor)' -> code='HORA_EXTRA'

'PAGTO. VALE-TRANSPORTE (Qtde)' + 'PAGTO. VALE-TRANSPORTE (Valor)' -> code='VALE_TRANSPORTE'

'PAGTO. VALE REFEICAO (Qtde)' + 'PAGTO. VALE REFEICAO (Valor)' -> code='VALE_REFEICAO'

'PREMIO/BONUS' -> code='PREMIO_BONUS'

'TRIBUTOS (VALOR)' -> code='TRIBUTO_VALOR'

'ENCARGOS (VALOR)' -> code='ENCARGO_VALOR'

'TAXA FATURAMENTO (VALOR)' -> code='TAXA_FATURAMENTO'

Se um valor for zero ou vazio, não criar o payroll_item.

No upload de exames:

Criar/associar unit pelo CNPJ da Unidade.

Criar/associar employee pelo CPF.

Determinar billing_period pelo mês de Dt.Exame.

Gravar em exam_record e também criar um payroll_item do tipo EXAME_MEDICO (amount = Vl.Cobrar R$).

Crie também um endpoint simples para listar todos os payroll_item de um billing_period agrupados por funcionário e somar amount por payroll_item_type.”
